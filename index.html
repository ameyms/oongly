(n) A nodejs based tool for uploading/downloading files and executing remote commands in non-homogeneous systems

Usage

Oongly comprises of two components:

oonglyd

This is a simple HTTP server that one needs to spawn on the machine where remote commands need to be executed or files are to be uploaded to or downloaded from

To start the oongly daemon, use this command: $ oonglyd --port 8080 Note that in above example --port is an optional argument and by default spawns the daemon on port 8080.

oonglycli

This is 'client' CLI which provides options to invoke oongly functionality on any remote machine running a oonglyd service

Using oonglycli, one can perform following tasks

Upload a file
$ oonglycli --host foo.local --port 8080 --op upload --file /home/ameyms/foo.sh --remotepath /var/fubar/bar.sh

Download a file
$ oonglycli --host foo.local --port 8080 --op download --file /home/ameyms/foo.sh --remotepath /var/fubar/bar.sh

Execute a command
$ oonglycli --host foo.local --port 8080 --op cmd cat /home/amey/code/*.js foo | wc -l

In all of the above examples, --host and --port are optional. --host defaults to localhost and --port defaults to 8080

NOTE: Oongly does not support authentication (yet). So be careful!

Install

You can install oonglycli as well as oonglyd via npm

oonglyd

To download and install oonglyd, run following command: $ npm install -g oonglyd

oonglycli

To download and install oonglycli, run following command: $ npm install -g oonglycli
